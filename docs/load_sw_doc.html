load_sw

Loads space weather data.

Back to Astrodynamics Toolbox Contents.

Syntax

[sw1,sw2] = load_sw
[sw1,sw2] = load_sw(MJD_UTC0,duration)

Description

[sw1,sw2] = load_sw loads the full space weather data sets.

[sw1,sw2] = load_sw(MJD_UTC0,duration) loads the space weather data sets with only the data needed for the simulations. "sw1" includes data from 3 days before the start of the simulation until the end of the simulation. "sw2" includes data from 5 days before the start of the simulation until the end of the simulation. The start date is specified as the modified Julian date of the initial UTC, and the end of the simulation is calculated using the simulation duration (duration, specified in hours).

Note

  • N = # of data entries for 1st half of space weather data.
  • M = # of data entries for 2nd half of space weather data.
  • The two data sets, sw1 and sw2, have data for different dates (specifically, the sw1 starts on an earlier date and ends on a later date than the sw2 data set). Thus, they must be loaded separately.
  • Even when only a subset of the dates in sw1 and sw2 are taken (based on the simulation start/end times), they still must be loaded separately because they require a different range of dates. Specifically, the parameters in sw2 are used for the Jacchia-Bowman 2008 model, and one of the parameters for this model requires a 5-day lag. However, for the parameters in the sw1 data set, the largest lag required is a 3-day lag.
